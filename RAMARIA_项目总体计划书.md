## 第一部分：项目定位与愿景

### 1.1 核心定位

**RAMARIA** 是一个以「记忆」为核心的个人 AI 陪伴系统，区别于传统 AI 助手每次对话从零开始的模式，它致力于构建一个**认识你、记住你、理解你**的虚拟伙伴关系。

| 维度 | 传统AI助手 | RAMARIA |
|------|-----------|---------|
| 学习内容 | 语气、风格 | 经历、情感、思维方式 |
| 记忆方式 | 无/每次重头 | 结构化分层存储 |
| 可解释性 | 黑盒 | 树状关联可回溯 |
| 隐私 | 数据上传 | 全量本地化 |

### 1.2 独特价值主张

> **用户不只是在使用AI，而是在培育一段记忆生态。每一次对话不是孤立事件，而是这片珊瑚礁的又一次生长。**

**珊瑚生态隐喻体系**将冰冷的AI交互转化为温暖、有机、可持续的共生体验：

| 生物特征 | RAMARIA隐喻 |
|---------|-------------|
| 菌丝网络结构 | 记忆节点形成非线性关联网络 |
| 菌丝共生关系 | 用户-AI记忆共享与协同成长 |
| 珊瑚分支生长 | 对话历史树状扩展 |
| 微光现象 | 安静时的记忆荧光，被动陪伴体验 |
| 孢子繁殖 | 任务创建与完成的生命周期 |

### 2.3 当前架构瓶颈

```
用户请求 → FastAPI → MemoryLayer → TaskRouter → LLM调用
           ↓                    ↓
    前端响应延迟        记忆检索效率低下（O(n)复杂度）
```

- **记忆检索**：顺序查找，O(n) 复杂度，随记忆增长性能下降
- **向量检索**：接口已就绪但未集成到对话流
- **实时推送**：无 WebSocket，状态更新依赖轮询
- **关联计算**：缺乏语义关联度算法，菌丝网络无数据支撑

### 4.2 关键技术突破

#### 突破1：记忆关联度计算引擎

```python
class MemoryAssociationEngine:
    """记忆关联度计算核心算法"""
    
    def calculate_association(self, memory_a: MemoryNode, memory_b: MemoryNode) -> float:
        # 1. 语义相似度（基于嵌入向量）
        semantic_sim = cosine_similarity(
            memory_a.embedding, memory_b.embedding
        )
        
        # 2. 时间衰减因子（近期记忆关联更强）
        days = (memory_b.created_at - memory_a.created_at).days
        time_factor = math.exp(-0.1 * days)
        
        # 3. 访问频率加权
        freq_factor = math.log(memory_a.access_count + memory_b.access_count + 1)
        
        # 4. 关键词重叠加成
        keyword_overlap = len(set(memory_a.keywords) & set(memory_b.keywords))
        keyword_factor = 1 + 0.2 * keyword_overlap
        
        return semantic_sim * time_factor * freq_factor * keyword_factor
```

#### 突破2：任务状态机与生命周期

```python
class SporeLifecycleManager:
    STATES = ['DORMANT', 'GERMINATING', 'GROWING', 'BLOOMING', 'DECOMPOSING', 'NUTRIENT']
    
    def advance_state(self, interaction: str = None) -> str:
        """孢子状态推进"""
        transitions = {
            'DORMANT': 'GERMINATING' if interaction else 'DORMANT',
            'GERMINATING': 'GROWING',
            'GROWING': 'BLOOMING' if self._check_deadline() else 'GROWING',
            'BLOOMING': 'DECOMPOSING',
            'DECOMPOSING': 'NUTRIENT'
        }
        self.state = transitions.get(self.state, self.state)
        return self.state
```

| 向量检索性能不足 | 高 | 高 | 分片索引、缓存预热、渐进加载 |
| WebSocket连接不稳定 | 中 | 中 | 断线重连、消息队列备份 |
| 3D可视化性能问题 | 高 | 中 | 2D回退模式、节点数量限制 |
| 记忆数据隐私泄露 | 低 | 极高 | 端到端加密、本地优先 |
| 用户接受度低 | 中 | 高 | 渐进引导、简化模式 |

### 7.2 项目管理风险

| 风险 | 概率 | 影响 | 缓解措施 |

### 7.3 应对策略

**技术风险**：
- 每项创新功能先做技术原型验证
- 预留 20% 时间缓冲给未知挑战
- 建立性能基准测试

**产品风险**：
- 每个阶段产出可用 MVP
- 用户反馈驱动迭代
- 保持核心对话功能简单稳定
