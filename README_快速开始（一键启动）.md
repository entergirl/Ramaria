## 前置准备

在使用珊瑚菌之前，你需要准备以下软件：

### 1. 本地推理服务（必须）

珊瑚菌依赖本地运行的 AI 模型来生成对话。你需要在启动珊瑚菌前先启动推理服务。

| 软件 | 下载地址 | 说明 |
|------|---------|------|
| **LM Studio**（推荐） | https://lmstudio.ai | 图形界面，易上手 |
| Ollama | https://ollama.com | 命令行，资源占用低 |

**LM Studio 启动步骤：**
1. 下载并安装 LM Studio
2. 在左侧切换到 **"Local Server"** 标签页
3. 搜索并加载一个对话模型（如 Qwen3.5、Gemma 等）
4. 点击 **"Start Server"** 按钮
5. 确认底部状态栏显示 **"Running"**

**Ollama 启动步骤：**
```bash
# 首次使用，拉取模型（以 qwen3.5 为例）
ollama pull qwen3.5:7b

# 启动服务
ollama serve
```

> ⚠️ **重要**：推理服务必须保持运行状态才能使用珊瑚菌。

### 2. 嵌入模型（必须）

嵌入模型用于将文本转换为向量，是珊瑚菌记忆检索功能的基础。

**推荐模型**：Qwen/Qwen3-Embedding-0.6B（约 600MB）

**下载方法**：

1. 访问国内镜像站下载（推荐）：
   https://hf-mirror.com/Qwen/Qwen3-Embedding-0.6B

2. 点击页面上的文件列表，下载**所有文件**

3. 将所有文件放入**同一个文件夹**，例如：
   ```
   D:/models/Qwen3-Embedding-0.6B/
   ├── config.json
   ├── model.safetensors
   ├── tokenizer.json
   └── ...
   ```

> 📝 **提示**：文件夹内必须包含 `config.json` 和模型文件，路径将在首次配置时使用。路径分隔符请使用正斜杠 `/`。

## 快速启动

### 步骤 1：启动珊瑚菌

1. 双击 **Ramaria.exe**
2. 首次运行自动进入配置向导（约 10-30 秒加载）

### 步骤 2：完成配置向导

按照配置向导的 5 个步骤填写：

| 步骤 | 配置项 | 说明 | 示例 |
|------|--------|------|------|
| 1 | 推理服务地址 | LM Studio 或 Ollama 的 API 地址 | `http://localhost:1234/v1/chat/completions` |
| 1 | 模型名称 | 推理服务中加载的模型名称（区分大小写） | `qwen/qwen3.5-9b` |
| 2 | 嵌入模型路径 | 之前下载的模型文件夹路径 | `D:/models/Qwen3-Embedding-0.6B` |
| 3 | 人格配置 | 可选，自定义珊瑚菌的性格和说话风格 | 默认即可跳过 |
| 4 | 可选配置 | 云端 API Key、天气城市、服务端口 | 均可跳过 |

> 📝 **提示**：步骤 1 和 2 为必填，步骤 3/4/5 均有合理默认值，可直接跳过。

### 步骤 3：开始使用

配置完成后，页面会自动跳转到聊天界面，开始与珊瑚菌对话！

### 后续修改配置

如果需要修改配置，通过系统托盘操作：

1. 右键点击任务栏托盘区的珊瑚菌图标
2. 选择 **"打开配置向导"**
3. 在向导中修改配置，保存后立即生效（无需重启）

## 数据与存储

运行后会在 Ramaria.exe **同级目录**创建以下文件夹：

```
Ramaria.exe  ← 主程序
├── .env           # 配置文件（由配置向导生成）
├── data/          # 数据库文件（对话记录、记忆等）
│   └── assistant.db
├── config/        # 配置文件
│   └── persona.toml
└── logs/          # 日志文件
    └── coral.log
```

### 数据迁移（旧版本升级）

如果你之前使用的是旧版本的珊瑚菌，可以迁移数据：

1. 找到旧版本的数据目录
2. 将其中的 `data` 文件夹内容复制到新版本目录
3. 首次启动会自动进行数据库迁移

### 完全卸载

删除 Ramaria.exe 所在文件夹即可。所有数据都存储在此文件夹内。

## 常见问题

### Q: 启动后显示"服务启动失败"或空白页面

请检查：
1. LM Studio 或 Ollama 是否已启动并加载模型？
2. 嵌入模型路径是否正确？（通过托盘菜单打开配置向导修改）
3. 端口 8000 是否被其他程序占用？
   - 运行 `netstat -ano | findstr :8000` 查看
   - 如被占用，在配置向导的可选配置步骤中修改服务端口

### Q: 配置向导中测试连接失败

1. 确认推理服务已启动并显示 "Running"
2. 确认模型名称与推理服务中加载的模型完全一致（区分大小写）
3. 浏览器访问 `http://localhost:1234/v1/models` 验证服务是否正常

### Q: 修改模型名称后仍然使用旧模型

配置向导保存后立即生效，无需重启。如果仍然使用旧模型，请确认：
1. 模型名称与推理服务中加载的模型名称完全一致
2. 在 LM Studio 的 Local Server 页面查看当前加载的模型标识符

### Q: 每次都要手动启动推理服务太麻烦

**LM Studio** 可以设置开机自启：
1. LM Studio 设置 → 通用 → 勾选"开机启动"
2. 勾选"启动时自动加载上次使用的模型"

**Ollama** 可以设置开机自启：
1. Windows 任务计划程序中添加 `ollama serve` 任务

### Q: 珊瑚菌回复很慢

1. 使用更小的模型（如 7B 而非 70B）
2. 在 LM Studio 中降低 GPU 加载层数
3. 关闭其他占用 GPU 的程序

### Q: 想清除所有对话记录

删除 `data/assistant.db` 文件，下次启动会自动创建新数据库。

### Q: 如何查看详细日志？

打开 `logs/coral.log` 文件查看运行日志。

### Q: 窗口关了但程序还在运行

珊瑚菌在系统托盘区运行。右键托盘图标可以：
- 点击"打开窗口"恢复界面
- 点击"退出"彻底关闭程序
